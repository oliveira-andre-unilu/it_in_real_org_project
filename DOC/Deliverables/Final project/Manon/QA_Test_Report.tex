\subsection{Unit \& Integration Tests}\label{unit-integration-tests}}

\hypertarget{unit-test-summary}{%
\subsubsection{Unit test summary}\label{unit-test-summary}}

During the execution of the unit tests, several issues were discovered,
mostly related to incorrect business logic in the backend layer. For
example, the verifyEmail() method returned false when the email did not
exist in the system, while logically it should only return false when
the email already exists. These issues demonstrate the usefulness of
unit tests in detecting logic defects very early in the development
phase.

Most modules passed unit tests successfully, including:

\begin{itemize}
\tightlist
\item
  Service layer
\item
  Domain and DTO objects
\item
  Exception handling
\item
  Security configuration
\end{itemize}

Overall, unit testing revealed multiple design inconsistency that need
to be fixed before release. The test campaign proved highly valuable for
improving code reliability, especially on core business logic.

\hypertarget{summary-of-detected-critical-errors}{%
\subsubsection{Summary of detected critical
errors}\label{summary-of-detected-critical-errors}}

Only one error was noted, concerning
TimestampEntryService.updateTimeStamp:

\begin{itemize}
\tightlist
\item
  the test reports a NonConformRequestedDataException related to
  Timestamp compliance, likely due to validation (duration / start time
  / business rule).
\item
  No other test revealed functional errors.
\item
  No errors related to the Repository or dependency injections.
\item
  Admin / self-access rules work correctly.
\item
  Business constraints (email, existing project, etc.) are properly
  enforced.
\end{itemize}

\hypertarget{conclusion}{%
\subsubsection{Conclusion}\label{conclusion}}

Unit tests demonstrate a stable, consistent, and secure application,
where:

\begin{itemize}
\tightlist
\item
  business rules are properly enforced,
\item
  administrative access controls are robust,
\item
  CRUD operations behave correctly,
\item
  exceptions are correctly propagated. The test coverage is extensive
  and relevant, covering both normal and exceptional scenarios. The only
  non-compliance to address concerns the update of a Timestamp,
  potentially related to overly strict business rule validation or a
  missing initialization in the mock.
\end{itemize}
